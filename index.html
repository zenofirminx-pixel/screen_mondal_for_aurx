Lead Engine & Compiler Architect

Concepteur originaire de la République Démocratique du Congo (RDC). Firmin a développé l'architecture VM AX3D pour s'affranchir du JavaScript classique et exécuter du bytecode ultra-compact directement dans le navigateur.

Pipeline de Compilation

1. Fichier .axs

Code source textuel à haut niveau (ex: object cube1 cube, cube1.color 255 0 0).

2. ax_compiler.py

Le script Python scanne les chaînes, convertit les tokens en opcodes numériques, et compacte les flottants (X, Y, Z, R, G, B) au format binaire standard IEEE 754.

3. Fichier .ax (Binaire)

Fichier de quelques octets s'ouvrant par le Magic Header 41 58 33 44 (AX3D) suivi des tables d'opcodes et des valeurs d'initialisation des structures.

Comment ce moteur lit le binaire :

Au lieu d'utiliser un interpréteur JS, ce moteur utilise l'API JavaScript DataView pour lire séquentiellement les octets du fichier binaire, instancier dynamiquement les géométries 3D nativas et lier l'input de l'utilisateur directement à la structure mémoire virtuelle.
